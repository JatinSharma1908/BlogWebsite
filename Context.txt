===========================================
BLOGWEBSITE - PROJECT CONTEXT
===========================================

PROJECT OVERVIEW:
-----------------
BlogWebsite is a premium Django-based blogging platform with a royal purple theme.
It features role-based access control (Authors & Readers), blog management, 
categories, tags, and a sophisticated user interface.

TECH STACK:
-----------
- Backend: Django 5.x (Python)
- Database: SQLite (development) - supports multi-tenancy
- Frontend: Bootstrap 5.3, Custom CSS (Royal Purple Theme)
- Icons: Bootstrap Icons
- Fonts: Google Fonts (Inter)

===========================================
DATABASE SCHEMA
===========================================

USERS TABLE (Custom User Model):
- id: Serial Primary Key
- name: VARCHAR(100)
- email: VARCHAR(150) UNIQUE (used as username)
- password: Hashed password
- is_active: BOOLEAN
- is_staff: BOOLEAN
- is_superuser: BOOLEAN
- created_at: TIMESTAMP

ROLES TABLE:
- id: Serial Primary Key
- name: VARCHAR(50) UNIQUE
  Available roles: 'Author', 'Reader'

PERMISSIONS TABLE:
- id: Serial Primary Key
- name: VARCHAR(100) UNIQUE

USER_ROLES TABLE (Many-to-Many):
- user_id: FK to users
- role_id: FK to roles

ROLE_PERMISSIONS TABLE (Many-to-Many):
- role_id: FK to roles
- permission_id: FK to permissions

CATEGORIES TABLE:
- id: Serial Primary Key
- tenant_id: INT (multi-tenancy support)
- name: VARCHAR(100)
- slug: VARCHAR(120) (auto-generated)
UNIQUE INDEX on (tenant_id, slug)

TAGS TABLE:
- id: Serial Primary Key
- tenant_id: INT
- name: VARCHAR(100)
- slug: VARCHAR(120)
UNIQUE INDEX on (tenant_id, slug)

BLOGS TABLE:
- id: Serial Primary Key
- tenant_id: INT
- title: VARCHAR(255)
- slug: VARCHAR(300) (auto-generated)
- excerpt: TEXT (optional)
- content: TEXT
- featured_image: TEXT (URL)
- status: VARCHAR(20) ('draft', 'published', 'archived')
- author_id: FK to users
- category_id: FK to categories
- published_at: TIMESTAMP
- created_at: TIMESTAMP
- updated_at: TIMESTAMP
INDEXES on (tenant_id, slug), (tenant_id, status)

COMMENTS TABLE:
- id: Serial Primary Key
- tenant_id: INT
- blog_id: FK to blogs
- name: VARCHAR(100)
- email: VARCHAR(150)
- comment: TEXT
- status: VARCHAR(20) ('pending', 'approved', 'spam', 'rejected')
- created_at: TIMESTAMP

===========================================
USER ROLES & PERMISSIONS
===========================================

READER ROLE:
- Browse published blogs
- Read blog content
- Post comments (pending approval)
- View own comment history

AUTHOR ROLE:
- All Reader permissions
- Create new blog posts
- Edit own blog posts
- Delete own blog posts
- Manage categories
- Manage tags
- Set blog status (draft/published/archived)
- Moderate comments on own blogs

ADMIN ROLE (Superuser):
- All permissions
- Access Django admin panel
- Manage users and roles
- Moderate all content

===========================================
WORKFLOW EXAMPLES
===========================================

NEW USER REGISTRATION:
1. Visit / (home page)
2. Click "Get Started" or "Register"
3. Fill registration form (name, email, role, password)
4. Role automatically assigned (Author or Reader)
5. Redirect to login
6. Login → Redirect to dashboard

BLOG CREATION (Author):
1. Login as Author
2. Dashboard → "Create New Blog" button
3. Fill form:
   - Title (required)
   - Excerpt (optional)
   - Content (required)
   - Category (select or create new)
   - Tags (comma-separated)
   - Featured image URL (optional)
   - Status (draft/published)
4. Submit → Redirect to "My Blogs"
5. View, edit, or delete from list

CATEGORY MANAGEMENT:
1. Dashboard → "Categories" or direct link
2. Left panel: Create new category form
3. Right panel: List of existing categories
4. Shows blog count per category
5. Auto-generates slug from name

===========================================
FILE UPLOAD NOTES
===========================================

Currently, featured images are handled via URL input.
To enable file uploads:
1. Install Pillow: pip install Pillow
2. Update models.py:
   featured_image = models.ImageField(upload_to='blog_images/')
3. Configure MEDIA settings in settings.py:
   MEDIA_URL = '/media/'
   MEDIA_ROOT = BASE_DIR / 'media'
4. Update urls.py to serve media files in development
5. Update forms.py widget to FileInput

===========================================
DEPLOYMENT CHECKLIST
===========================================

1. Security:
   - Set DEBUG = False
   - Configure ALLOWED_HOSTS
   - Use environment variables for SECRET_KEY
   - Set up HTTPS
   - Configure CSRF_TRUSTED_ORIGINS

2. Database:
   - Migrate to PostgreSQL for production
   - Set up database backups
   - Configure connection pooling

3. Static Files:
   - Run python manage.py collectstatic
   - Configure WhiteNoise or CDN
   - Minify CSS/JS

4. Performance:
   - Enable caching (Redis/Memcached)
   - Set up database indexes
   - Configure gunicorn/uwsgi
   - Enable gzip compression

5. Monitoring:
   - Set up error logging (Sentry)
   - Configure application monitoring
   - Set up health checks

===========================================
TROUBLESHOOTING
===========================================

ISSUE: Categories not showing in dropdown
FIX: Create categories first via /categories/

ISSUE: User can't create blogs
FIX: Ensure user has 'Author' role assigned

ISSUE: Static files not loading
FIX: Check STATIC_URL and STATICFILES_DIRS in settings.py
     Run python manage.py collectstatic

ISSUE: Database locked error
FIX: Close all connections to SQLite database
     Restart development server

ISSUE: Form validation errors
FIX: Check browser console for JavaScript errors
     Verify form field requirements
